# Phishing Email Investigation Lab

## Project Description

This project is a hands-on phishing email investigation lab created to practice SOC analyst email triage skills. The goal of this project is to analyze suspicious email samples, review message headers, identify spoofing indicators, investigate suspicious URLs/domains, and document findings in a clear incident-style format.

Phishing investigation is a common task for entry-level cybersecurity analysts and SOC analysts. This project demonstrates my ability to review suspicious emails, identify indicators of compromise, and communicate findings clearly.

## Why I Created This Project

I created this project as part of my cybersecurity transition and portfolio development. After completing network traffic analysis projects with Wireshark, I wanted to build another practical project focused on email security and phishing investigation.

This project helps demonstrate hands-on skills in email header analysis, social engineering detection, threat intelligence lookup, IOC extraction, and SOC-style reporting.

## Investigation Methodology

For each phishing email case, I followed this process:

1. Reviewed the email subject, sender, and message content.
2. Checked for social engineering indicators such as urgency, fear, fake login requests, or payment pressure.
3. Reviewed sender, Reply-To, and Return-Path fields.
4. Analyzed email headers using header-analysis tools.
5. Checked SPF, DKIM, and DMARC authentication results.
6. Investigated suspicious URLs or domains using threat intelligence tools.
7. Extracted indicators of compromise.
8. Determined a final verdict.
9. Documented recommended response actions.
